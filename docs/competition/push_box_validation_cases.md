# Push Box 算法验证用例

本文记录用于验证 RT1064 推箱子 BFS 的最小用例。用途是先用 SokoPlayer HTML5 的 XSB/LURD 格式建立标准答案，后续再让 RT1064 输出路径并逐项对比。

在线验证工具：

```text
https://sokoban.cn/sokoplayer/SokoPlayer_HTML5.php
```

## 格式约定

### SokoPlayer XSB 格式

```text
# 墙
  空地
@ 玩家
$ 箱子
. 目标
```

### LURD 答案格式

```text
l/r/u/d 普通移动
L/R/U/D 推箱移动
```

例如 `rR` 表示：先向右走一步，再向右推箱一步。

### RT1064 离线地图格式

```text
# 墙
. 空地
C 车
B 箱子
T 目标
```

## 使用方法

1. 打开 SokoPlayer HTML5。
2. 将某个用例的 `XSB` 内容粘贴到网页底部文本框。
3. 点击 `载入关卡`。
4. 将同一用例的 `Expected LURD` 粘贴到文本框。
5. 点击 `载入答案`。
6. 若网页显示 `Solution loaded.`，说明该答案格式可被工具接受。
7. 后续 RT1064 BFS 输出应与 `Expected LURD` 等价，至少应保证箱子最终到达目标。
